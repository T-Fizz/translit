translit

A small Python library for name-aware transliteration. Eight language pairs. github.com/T-Fizz/translit

Install

pip install -e git+https://github.com/T-Fizz/translit.git#egg=translit-core

Use

from translit_core import transliterate

transliterate("カナちゃん", "en")                              # → "Kana-chan"
transliterate("山田太郎", "en", source_lang="ja")              # → "Yamada Taro"
transliterate("山田太郎", "en", source_lang="ja",
              name_order="given-first")                      # → "Taro Yamada"
transliterate("김정은", "en", source_lang="ko")                # → "Kim Jeong-eun"
transliterate("Иван Сергеевич Тургенев", "en")                # → "Ivan Sergeevich Turgenev"
transliterate("नरेंद्र मोदी", "en", source_lang="hi")           # → "Narendra Modi"
transliterate("عبد الرحمن", "en", source_lang="ar")            # → "Abdul-Rahman"
transliterate("ทักษิณ", "en", source_lang="th")               # → "Thaksin"
transliterate("John Smith", "ja")                              # → "ジョン・スミス"

Returns None when the engine can't deterministically romanize an input. Fail-soft: callers see a clear signal, never a half-translated guess.

Supported pairs

Source Target Method
Japanese (kana + kanji) Latin pykakasi passport-style + 22 honorifics + reverse-katakana round-trip
Chinese (Hanzi) Latin pypinyin (no tones)
Korean (Hangul) Latin RR + 36-name surname overlay + 3 honorifics
Russian (Cyrillic) Latin BGN/PCGN press-style
Hindi (Devanagari) Latin IAST + diacritic strip + schwa deletion
Arabic Latin Curated 50-name overlay
Thai Latin RTGS via pythainlp
English (Latin) Japanese (katakana) alkana + acronym fallback

Examples

ja → en honorifics passport hepburn

  • カナちゃん → Kana-chan
  • 山田太郎 → Yamada Taro
  • 田中先生 → Tanaka-sensei
  • 兄ちゃん → Niichan
  • ヴィクター → Victor round-trip via reverse-alkana

ko → en surname overlay press style

  • 김정은 → Kim Jeong-eun
  • 박지성 → Park Ji-seong
  • 문재인 → Moon Jae-in
  • 남궁민 → Namgoong Min 2-syllable family
  • 박지성씨 → Park Ji-seong-ssi

ru → en BGN/PCGN

  • Иван Сергеевич → Ivan Sergeevich
  • Михаил Ахматова → Mikhail Akhmatova х → kh, not h
  • Фёдор Семён → Fyodor Semyon ё-glide preserved
  • Ольга Объект → Olga Obyekt

hi → en schwa deletion

  • अमित → Amit
  • कृष्ण → Krishna cluster keeps schwa
  • अमिताभ बच्चन → Amitabh Bachchan
  • सुनीता → Sunita long ā kept distinct from schwa
  • नरेंद्र मोदी → Narendra Modi

ar → en curated overlay

  • محمد → Muhammad
  • فاطمة → Fatima
  • عبد الرحمن → Abdul-Rahman multi-token compound
  • مُحَمَّد → Muhammad tashkeel stripped

th → en RTGS

  • สมชาย → Somchai
  • ทักษิณ → Thaksin
  • ประยุทธ์ → Prayut
  • เพื่อน → Phuean vowel reordered around the consonant

en → ja alkana acronym fallback

  • John → ジョン
  • John Smith → ジョン・スミス
  • Mary-Jane → メアリー・ジェイン
  • Mr. Smith → ミスター・スミス
  • FBI → エフビーアイ letter-by-letter for unknown acronyms

What it handles

  • Japanese honorifics including compounds (ちゃん, 先生, 兄ちゃん, にゃん)
  • CJK source disambiguation (田中 → Tanaka if ja, Tian Zhong if zh)
  • Multi-kanji morpheme splitting (山田太郎 → "Yamada Taro")
  • Round-trip katakana → Western name (ヴィクター → "Victor")
  • Korean traditional surname spellings (Kim/Lee/Park, not Gim/I/Bak)
  • Korean 2-syllable family names (남궁/황보/사공/제갈/선우/독고)
  • Russian press-style (Mikhail not Mihail, Fyodor not Fedor, ъе/ье → ye)
  • Hindi schwa deletion with cluster + aspirated-digraph awareness
  • Arabic input normalization: tashkeel, alif variants, tatweel, Allah ligature
  • Half-width katakana / full-width Latin (NFKC)
  • Punctuation stripping, silent-character dropout detection
  • Family-first vs given-first name ordering (name_order=)

Why a library, not just a service

An in-process call is ~3μs warm. An HTTP roundtrip is at least four orders of magnitude slower. If your consumer is Python, just import the library directly. The repo also ships a FastAPI service (app/) for the case where the consumer isn't Python or where multiple apps need a shared cache.
